# E-commerce Product Analytics Project
## 📊 Objective
Identify conversion bottlenecks in the e-commerce funnel to improve user experience and increase revenue.
🚀 Analyzed 1M+ user events to identify conversion bottlenecks in an e-commerce funnel

## 📁 Dataset
- 1M+ user interaction records
- Dataset Source: Kaggle (E-commerce behavior dataset)
- Events: view, cart, purchase

## 🔧 Approach  
- Cleaned and structured raw event data  
- Used SQL to analyze user-level behavior  
- Built funnel metrics (View → Cart → Purchase)  
- Identified drop-offs and conversion bottlenecks 

## 🔍 Key Analysis
### Funnel Analysis
- View Users: 908
- Cart Users: 168
- Purchase Users: 18

- ### Conversion Rates
- Cart Conversion: ~18.5%
- Purchase Conversion: ~2%

## 🚨 Key Insights
- ~98% drop-off from product view to purchase
- Low cart conversion indicates weak product engagement
- Significant drop during checkout phase

## 💡 Recommendations
- Improve product page UX (images, reviews, descriptions)
- Simplify checkout process
- Implement cart abandonment strategies
- Run A/B tests on UI and pricing

## 📈 Business Impact
- Improving conversion from 2% to 3% can lead to ~50% increase in revenue  
- Identified key friction points affecting user journey  
- Insights can guide product and UX improvements  

## 🛠 Tools Used
- SQL (SQLite)

## 🎯 Outcome
Identified major conversion bottlenecks and proposed data-driven improvements.
